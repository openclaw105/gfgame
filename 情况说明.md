## 4. 已知结构问题（重点）

### 4.1 第 4 章：选项顺序 vs 主线 mid 错位

生成器使用固定节拍 `CHOICE_BEAT_POSITIONS = [7, 14, 21, 28, 32, 35]`：先播 OPEN，再按槽位插入 MID，再在指定 beat 插入选项。

**第 4 章实际播放顺序（简化）**：

```text
OPEN（4 句）→ 保安拦门
MID ×2      → 薇薇：通行证在二号桌 / 葵：在门口已被拦
【选项 C1】门口被拦 ── 含「联系宋威龙助理」  ← 此时尚未见过宋威龙
MID 后续    → 进大厅 → 卢昱晓 → 宋威龙递名片 → 肖战敬酒 → 花园…
【选项 C2】宋威龙递名片                      ← 名片已在 mid 演过，题干重复
【选项 C3】肖战敬酒                         ← 同上
…
```

**结论**：

1. **「联系宋威龙助理」在叙事上无铺垫**：OPEN/MID（C1 前）未出现宋威龙或「工作室可补登记」类台词。
2. **C2/C3 与 mid 内容重复**：玩家先看完 mid 里的递名片、敬酒，再被问「递名片你？」「敬酒你？」。
3. **C1 默认分支异常**：`CH04.branch = {}`，走 `branch_for()` 时，第 4 章 `SPEAKERS` 为卢昱晓、宋威龙、葵（**无薇薇、无保安**），导致「出示通行证」由卢昱晓接话等违和台词。

### 4.2 默认分支模板：「记进心里」

位置：`story_content.py` → `branch_for()` 收束句之一：

```python
f"手机屏幕暗下去，「{short}」这句决定被你记进心里。"
```

- 当前 `data/chapters.json` 中 **约 22 处**「记进心里」。
- **第 4、6 章**因 `branch: {}`，**所有选项**均可能落入该模板 + 轮换式「XX听完…把话题挡了一下」。
- 用户反馈：不要每次选完就「默默记在心里」，应改为**现场对白**（与第 1、2、3、5 章 `CHxx_BRANCH` 一致）。

### 4.3 第 6 章：张艺兴线铺垫不足

- OPEN 直接「啤酒节后台 + DEMO」，未说明葵为何在场（薇薇派活 / 控台记色卡等）。
- 与第 5 章片场、第 7 章李昀锐之间缺一句「啤酒节统筹」类过渡。

### 4.4 第 5 章：片场开场

- OPEN：「宋威龙助理来接你」——若第 4 章选「提前告辞」且几乎未互动，仍进片场；宜加「按会所名片约探班」一句。

## 7. 引擎机制摘要

| 机制 | 说明 |
|------|------|
| `requires_branch_tag` | 选项/支线需匹配本章分支 tag（如第 2 章兑奖需 `buy`） |
| `shouldSkipTaggedBeat()` | 未满足 tag 时跳过该 beat |
| `choiceOptionsForBeat()` | 过滤不可选选项 |
| 机缘卡 | 第 10 章等可使用，影响结局与数值 |
| 数值维度 | `xz_love` / `xz_trust` / `swl_love` / `zyx_love` / `lyr_love` / `vv_friend` / `vv_gj` / `art_value` / `public_opinion` / `misunderstanding` 等 |
