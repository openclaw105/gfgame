游戏类型：网页视觉小说乙游，单周目目标时长约60到90分钟，支持多周目收集称号、结局图鉴和机缘卡路线。
核心叙事：玩家扮演葵，在欧洲画展与肖战偶遇，回国后围绕艺术工作室、明星活动、片场、会所聚会、闺蜜薇薇的单位后勤工作展开情感与事业双线。
核心结构：10章。第1-2章发散，第3章收束；第4-5章发散，第6章收束；第7-8章发散，第9章收束；第10章按变量判定9种结局。

3.1 金钱与机缘卡判定
每次奇遇或剧情节点改变 money 后调用 checkMoneyPrompt()。重点规则：只要存款跨过 next_card_prompt，就弹出一次“是否兑换机缘卡”。玩家选择兑换则 money -= 30，chance_card += 1；玩家选择暂不兑换则保留金钱，并把 next_card_prompt 提升到下一个30万档，避免同一档反复弹窗。
function addMoney(delta, reason){
  money = Math.max(0, money + delta);
  money_peak = Math.max(money_peak, money);
  logMoney(reason, delta, money);
  checkMoneyPrompt();
}
function checkMoneyPrompt(){
  while(money >= next_card_prompt){
    showCardPrompt(next_card_prompt);
    next_card_prompt += 30;
  }
}
function exchangeChanceCard(){
  if(money >= 30){ money -= 30; chance_card += 1; showToast("获得机缘卡x1"); }
}

十五、扇形发散与收束剧情执行模型
本章节定义整部游戏的核心剧情路由。发散不是简单的三选一短回应，而是纵向树状剧情：每一轮发散起点先进入3条一级路线，每条一级路线再分裂为3条二级路线，共9种走向；在收束章之前，玩家实际看到的事件、背景、出场角色、好感变动、奇遇任务和称号均可以完全不同。收束发生时，剧情主事件归一，但系统保留所有分支触发记录、变量数据、称号、金钱和机缘卡使用情况。
发散起点命名为 Dn-1，例如第一轮发散为 D1-1。玩家选择A/B/C后进入剧情2/3/4，即一级路线。
一级路线中的关键选择再次分裂，每条产生3条二级路线，合计剧情5到13，即九宫格路线。
收束章读取 branch_history、route_flag、affinity、trust、money、chance_card、public_heat、art_value、vv_friend、vv_gj 等变量，生成不同的收束台词与奖励，但下一轮发散的起点统一。
程序上建议把每条分支写为 node_id。主线节点只负责归一，分支节点负责短期剧情体验和变量沉淀。
张艺兴素材库中存在跳舞图片，统一注册为 char_张艺兴_跳舞。张艺兴舞蹈彩蛋概率高，默认基础概率45%；若拥有“DEMO受害者”称号或trust_zyx>=30，概率提升到75%；若玩家连续拒绝听DEMO三次，概率降至20%。
