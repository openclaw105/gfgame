# 剧情合理化优化执行文档

生成日期：2026-05-24

用途：交给其他 agent 执行剧情优化。本文只定义改稿目标、剧情方案、验收标准，不直接修改游戏剧情源。

## 一、执行原则

### 1. 只改剧情源，不手改生成物

主要修改：

- `scripts/story_content.py`

改完必须执行：

```powershell
cd D:\k\GFgame
python scripts/generate_chapters.py
python scripts/bundle_offline_data.py
python scripts/validate_story.py
```

不要直接手改：

- `data/chapters.json`
- `js/offline-data.js`
- `docs/剧情设计说明.md`

这些应由脚本重新生成或另行导出。

### 2. 明星登场必须有“职业理由”

每个明星出场前，至少满足一个理由：

- 受品牌、活动、剧组、工作室邀请。
- 与当前场景有工作任务。
- 与上一章事件有明确承接。
- 由薇薇、龚俊、宋威龙、肖战等已在场角色合理引荐。

禁止写法：

- “某某刚好路过。”
- “某某突然出现。”
- “某某看了你一眼没说话。”
- “你默默记在心里。”
- “他/她没有追问，只把下一步交回现场。”

### 3. 选择后必须有真实社交反馈

每个选项分支至少 3 句，建议结构：

1. 女主明确行动或说出口。
2. 被影响的人即时回应。
3. 场景出现一个小变化，回到主线。

可接受示例：

```python
[
    ("葵", "我先把艺人通道清出来，采访往右侧引。"),
    ("欧阳娜娜", "可以，我让助理通知摄影别堵门。"),
    ("narrator", "通道安静下来，肖战终于能从彩排区退到后台。"),
]
```

不可接受示例：

```python
[
    ("葵", "我先处理。"),
    ("欧阳娜娜", "行，按你说的来。"),
    ("narrator", "手机屏幕暗下去，这句决定被你记进心里。"),
]
```

## 二、必须上场的三个女明星

本次优化把“三个女明星”定义为：

- 卢昱晓：已在第 4 章上场，但入场和选项分支需要更合理。
- 欧阳娜娜：有素材，当前未进入剧情。
- 田曦薇：有素材，当前未进入剧情。

这三人应形成一条“品牌活动女明星组”支线，和第 3 章布场、第 4 章会所自然衔接。

### 角色功能定位

| 角色 | 建议功能 | 首次合理登场 |
|---|---|---|
| 欧阳娜娜 | 品牌音乐合作嘉宾，懂舞台动线和采访节奏 | 第 3 章品牌活动布场 |
| 田曦薇 | 品牌短片女主/站台嘉宾，负责拍摄物料 | 第 3 章品牌活动布场 |
| 卢昱晓 | 会所聚会里的影视圈朋友，与宋威龙同桌 | 第 4 章会所 |

素材建议：

- 欧阳娜娜：开心、傲慢、伤心、愤怒。
- 田曦薇：开心、傲慢、悲伤、愤怒。
- 卢昱晓：开心、傲慢、悲伤、愤怒。

执行时应避免继续生成 `char_欧阳娜娜_一般`、`char_田曦薇_一般`、`char_卢昱晓_一般`，因为实际素材没有“一般”表情。
