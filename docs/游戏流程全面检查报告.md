## 3. 剧情走向与逻辑冲突

### 问题 A：大量分支仍是模板化补句

来源：

- `scripts/story_content.py:1273` 的 `branch_for(...)`
- `scripts/story_content.py:1282` 固定生成“行，按你说的来。”
- `scripts/story_content.py:1283` 固定生成“话头落下，各自继续忙手里的事。”

扫描结果：

- 共发现 129 处选项分支包含模板化回应。
- 这类分支在流程上不冲突，但剧情质感会明显重复。
- 对“每章丰富、每个章节不少于 30 次对话 6 次选择”的目标来说，数量达标，内容丰富度仍不足。

建议：

- 保留 `branch_for` 作为兜底，但优先为 60 个主选项补真实分支台词。
- 特别优先补第 4-10 章，因为模板化分支集中出现在中后段。

### 问题 B：`reject_demo_count` 不会生效

来源：

- `scripts/story_content.py:784` 写入 `{"reject_demo_count": 1}`
- `js/engine.js:63` 默认状态包含 `reject_demo_count`
- `js/engine.js:780` 的张艺兴跳舞彩蛋会读取 `reject_demo_count`
- 但 `js/engine.js:230` 的 `applyEffects(...)` 没有应用该字段

影响：

- 玩家拒绝 DEMO 的次数不会累计。
- `reject_demo_count >= 3` 的概率惩罚逻辑实际不可触发。

建议：

- 在 `applyEffects` 中补充 `reject_demo_count` 的数值累加。

### 问题 C：第 10 章机缘卡选项会绕过资源校验

来源：

- `scripts/story_content.py:1224`：用给肖战，`chance_card: -1`
- `scripts/story_content.py:1225`：用给宋威龙，`chance_card: -1`
- `js/engine.js:249`：直接 `s.chance_card += eff.chance_card`
- `js/engine.js:1054` 的 UI 机缘卡使用函数有数量校验，但剧情选项没有

影响：

- 玩家没有机缘卡时，也可能通过剧情选项白嫖好感。
- `chance_card` 可能变成负数。

建议：

- 给剧情选项增加条件禁用逻辑，或把第 10 章这两个选项改为调用统一的 `useChanceCard` 机制。

### 问题 D：部分结局数值不可自然到达

按剧情选择可达区间粗算：

| 字段 | 剧情内范围 |
|---|---:|
| money | 0-133 |
| swl_love | 最高约 40 |
| zyx_love | 最高约 18 |
| vv_gj | 最高约 42 |
| public_opinion | 最高约 66 |
| misunderstanding | 最高约 41 |
| vv_friend | 最低约 40 |

受影响结局：

- `雨停之后，无人归来`：要求 `misunderstanding >= 80 && vv_friend <= -20`，当前剧情自然不可达。
- `听见月光在起舞`：要求 `zyx_love >= 65 && zyx_dance_seen`，当前剧情最高约 18，过度依赖机缘卡与随机彩蛋。
- `风从片场吹来`：要求 `swl_love >= 75`，当前剧情最高约 40，需要机缘卡补足。
- `花开两岸，人间重逢`：要求 `vv_gj >= 70`，当前剧情最高约 42，需要机缘卡补足。
- `灯火两端，旧梦不言` 的显式条件要求 `public_opinion >= 85 && art_value < 55`，当前舆论最高约 66；但它仍可能作为 fallback 出现。

建议：

- 为每个结局设计一条明确可达路线。
- BE 路线需要增加误会值、降低薇薇友情的选项。
- 宋威龙、张艺兴、龚俊相关结局需要提高对应角色剧情选项的数值收益，或下调结局阈值。

实测视口：390 × 844。

- 背景加载正常。
- 人物与对话框无重叠。

- 三个选项按钮均被检测为存在 overflow 风险，虽然当前短文本肉眼上不一定明显，但长选项更容易出现拥挤。

- `css/main.css:1167`：人物舞台底边对齐逻辑
- `css/main.css:1327`：底部栈
- `css/main.css:1378`：选项面板

- `css/main.css:1497`：选项按钮

建议：

- 有选项时增加舞台底部预留，或下压/缩小立绘。
- `.game-ui.has-choices` 状态下单独降低 `.sprite-wrap` 的最大高度。
- 选项按钮中的字母标签和文本之间建议明确分离，避免 `A一起举杯` 这种连在一起的视觉。
- 长选项应允许自然换行，并避免按钮内部 overflow 判定为 true。

## 6. 优先修复顺序

1. 修 `applyEffects`：补 `reject_demo_count`，并保护 `chance_card` 不为负。
2. 修第 10 章机缘卡选项：没有机缘卡时禁用或改文案。
3. 修结局可达性：每个结局补一条可达路线。
4. 修 `branch_for` 模板化分支：优先替换 129 处“行，按你说的来”。
5. 修人物表情映射：减少 543 次兜底。
6. 修手机选项态遮挡：重点处理 `.game-ui.has-choices`。
7. 补用未使用背景、BGM、人物素材。

## 7. 总结

游戏当前可以运行，主线结构、章节顺序、基础素材加载没有硬阻断。

真正的问题集中在三类：

- 数值逻辑：部分字段不生效，部分资源可被绕过，部分结局不可达。
- 内容丰富度：大量分支仍是模板句。
- 素材精度：背景/BGM能用，但人物表情大量兜底，未完全贴合剧情。

建议先修逻辑，再扩写分支，最后补全素材表现。
